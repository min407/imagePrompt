❌ 常见错误和解决方案

错误1: Python未找到

现象: 运行代码时提示 "python: command not found"

解决:
• Windows: 重新安装Python时勾选"Add Python to PATH"
• Mac: 使用 brew install python3
• 在VS Code中按 Ctrl+Shift+P → 输入"Python: Select Interpreter"

错误2: 缩进错误

现象: "IndentationError: unexpected indent"

解决: Python用缩进代替大括号，确保缩进一致（推荐4个空格）

错误3: 中文乱码

现象: 输出中文时显示乱码

解决: 文件开头添加 # -*- coding: utf-8 -*-

🤖 AI应用开发流程科普

1. 需求分析

这是什么: 明确你的AI应用要解决什么问题

为什么重要: 避免开发无用功能，节省时间和成本

例子: "我需要一个能自动总结会议记录的工具"

2. 选择AI模型

这是什么: 根据需求选择合适的AI模型

为什么重要: 不同模型擅长不同任务，选择错误的模型效果会很差

模型选择指南:
• 文本处理 → GPT-3.5/4, Claude
• 图像识别 → GPT-4V, DALL-E
• 代码生成 → Codex, Copilot

3. API集成

这是什么: 将AI服务连接到你的应用

为什么重要: 这是AI应用的技术核心，决定了如何调用AI能力

常见AI服务商:
• OpenAI (ChatGPT)
• Google (Gemini)
• 百度 (文心一言)
• 阿里 (通义千问)

4. 数据处理

这是什么: 准备和处理输入/输出数据

为什么重要: 垃圾进垃圾出，数据质量决定AI效果

数据处理步骤:
• 数据清洗 → 去除无用信息
• 格式转换 → 统一数据格式
• 结果存储 → 保存AI返回结果

5. 用户体验设计

这是什么: 设计友好的用户界面

为什么重要: 好的体验让用户愿意使用你的应用

用户体验要点:
• 加载状态显示 (AI处理需要时间)
• 错误提示友好
• 结果展示清晰

6. 测试和优化

这是什么: 测试应用并优化性能

为什么重要: 确保应用稳定且性能良好

测试重点:
• 功能正确性
• 响应速度
• 错误处理

步骤2: 如何给变量命名？

变量命名规则（非常重要！）：

  • 只能用字母、数字、下划线
  • 不能用数字开头
  • 区分大小写（name 和 Name 不同）

    "smart": "gpt-4",             # 智能、昂贵
    "balanced": "gpt-4-turbo"      # 平衡选择
}

# 📊 Token配置 (成本控制)
TOKEN_LIMITS = {
    "short_answer": 100,      # 简短回答 ~0.02元
    "medium_answer": 500,     # 中等回答 ~0.1元  
    "long_answer": 1500,      # 详细回答 ~0.3元
    "max_context": 4000       # 最大上下文长度
}

# 🌡️ Temperature配置 (创造性控制)
TEMPERATURE_SETTINGS = {
    "factual": 0.2,           # 事实性回答
    "balanced": 0.7,          # 平衡模式
    "creative": 1.2           # 创造性模式
}

# 💰 成本预估 (每1000 tokens的价格)
COST_PER_1K_TOKENS = {
    "gpt-3.5-turbo": 0.002,  # 0.2分
    "gpt-4": 0.03,            # 3分
    "gpt-4-turbo": 0.01       # 1分
}

# 🎯 当前配置
current_config = {
    "model": AVAILABLE_MODELS["balanced"],
    "max_tokens": TOKEN_LIMITS["medium_answer"],
    "temperature": TEMPERATURE_SETTINGS["balanced"],
    "api_key": API_KEY
}

# 💡 计算单次调用成本
def estimate_cost(config, tokens_used):
    model = config["model"]
    cost_per_1k = COST_PER_1K_TOKENS.get(model, 0.01)
    cost = (tokens_used / 1000) * cost_per_1k
    return cost

print(f"最大Token: {current_config['max_tokens']}")

            plan[f"Day {i+1}"] = current_topics[i]
